Тип интерфейса приложения

Определяет технологический стек и пользовательский опыт

A

Десктопное приложение с GUI

Полноценное окно: дашборд, история сканирований, настройки. Иконка в трее. Как Malwarebytes или Kaspersky.

Плюсы

  • Удобно для обычных пользователей
  • Красивый UI, можно выложить публично
  • Полный контроль над UX

Минусы

  • Больше работы по фронтенду
  • Нужен фреймворк (Electron / .NET WPF / Tauri)
B

Лёгкий трей + CLI

Минималистично: иконка в трее, уведомления об угрозах, вся настройка через терминал или конфиг-файл. Для продвинутых пользователей.

Плюсы

  • Быстро разработать
  • Мало ресурсов
  • Хакерский шарм

Минусы

  • Не для массовой аудитории
  • Сложнее публиковать как продукт
C

Локальный веб-интерфейс

Бэкенд-сервис на Windows, UI в браузере (localhost). Современный стек, красивый интерфейс, кросс-платформенный UI.

Плюсы

  • Современный красивый UI
  • Любой веб-стек
  • Легко добавить удалённый доступ

Минусы

  • Нужен запущенный браузер
  • Не нативный опыт
